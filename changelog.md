# Changelog

This file contains all the merge commits, ordered from the first one until the last one.

## Working Process Overview

The development and validation process of this SRS followed the steps below:

1. The team conducted three meetings to validate the SRS document.
2. Suliman distributed the tasks of rephrasing and correcting the defects identified during the meetings.
3. Each group member worked on their own branch in GitHub. The branch named `project-part-2` belonged to the team leader.
4. Any assigned task was to be pushed to the branch of the person in charge.
5. After a push was made, the person in charge had to notify the team leader to merge their branch into the `project-part-2` branch.
6. Before pushing a new commit, each team member had to check the `project-part-2` branch for updates. If there were any changes, they needed to incorporate them first to avoid merge conflicts.  
   Ideally, this should have been done using the `git pull` command. However, the team was not very familiar with Git workflows, and there was not enough time to go through the learning process.
7. Once the team leader was informed, they performed the merge using the `pull request` feature in GitHub, which was then automatically reflected in the `project-part-2` branch.
